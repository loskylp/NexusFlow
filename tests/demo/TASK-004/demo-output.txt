Note: TASK-004 covers Redis Streams internals. Direct redis-cli access is not
available from outside the staging cluster. Redis Streams behavior is verified
IMPLICITLY through the API-level task lifecycle: submitting a task via the API
enqueues it to a Redis stream, and a worker consumes it via XREADGROUP.

The scenarios below demonstrate that the Redis Streams infrastructure is
functioning correctly by exercising the full task lifecycle through the API.
